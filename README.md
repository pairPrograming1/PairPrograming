PairProgramming - Resumen técnico

Principalmente usamos:
- JavaScript / TypeScript (front y back) y Python.

Frontend (ejemplos):
- React / Next.js (App Router) — principal.
- Vue.js y Angular — cuando aplica.
- Mobile: React Native.

Backend:
- Node.js / TypeScript (Express, Fastify, Next API routes) — principal.
- Python (FastAPI, Flask) — muy usado para ML, APIs y scripts.

Hosting y plataformas (preferidas):
- Vercel — despliegue frontend / fullstack moderno (Next.js).
- AWS — para infraestructura, servicios gestionados y escalado.
- Railway — despliegues rápidos y sencillos para prototipos.

Bases de datos:
- PostgreSQL — preferida para datos relacionales.
- MongoDB — cuando necesitamos flexibilidad de esquemas.
- MySQL — compatibilidad / proyectos legacy.

DevOps / entrega:
- CI/CD (GitHub Actions, GitLab CI) — sí, siempre integrado.


Otras consideraciones:
- Infra y deploys serverless (Vercel, AWS ) para reducir operaciones.
- Prefiero TypeScript donde es viable por seguridad y DX.
- Para ML/IA: Python + infra en cloud (GPU/TPU) o microservicios en Node/Python según el caso.

Cómo ejecutar (rápido):
1. Instala dependencias:
	npm install
2. Levanta el servidor en modo desarrollo:
	npm run dev
